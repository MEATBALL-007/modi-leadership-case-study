# นเรนทรา โมดี — Interactive Kinetic Presentation

พรีเซนต์เว็บหน้าเดียว (single-page) แบบ kinetic/animated สำหรับนำเสนอกรณีศึกษาผู้นำ
**นเรนทรา โมดี (Narendra Modi)** นายกรัฐมนตรีอินเดีย — เนื้อหาภาษาไทย โทนวิชาการ/มืออาชีพ
นำเสนอทั้งด้านบวกและด้านลบอย่างเป็นกลาง

> **ผลลัพธ์:** `index.html` — ไฟล์ **self-contained** เปิดด้วยการดับเบิลคลิก (`file://`) ได้ทันที
> ไม่ต้องต่อเน็ต ไม่มี CDN ไม่มี build step ตอนเปิด (ฟอนต์ไทย/ไอคอน/ภาพ ฝังในไฟล์ทั้งหมด)

## เปิดใช้งาน

ดับเบิลคลิก **`index.html`** ได้เลย — ใช้งานได้ออฟไลน์ 100%

### คีย์ลัด / ปุ่มควบคุม

| ปุ่ม | การทำงาน |
|------|----------|
| `M` | สลับโหมด **เลื่อน (Scroll) ↔ สไลด์ (Slide)** |
| `←` `→` `Space` | เลื่อนสไลด์ (เฉพาะโหมดสไลด์) · ปัด (swipe) บนทัชสกรีนได้ |
| `Home` / `End` | ไปสไลด์แรก / สุดท้าย |
| `S` หรือ `N` | เปิด/ปิด **โน้ตผู้พูด** (ผู้ชมไม่เห็น) |
| `A` | เปิด/ปิด **อนิเมชัน** (เคารพ `prefers-reduced-motion` อัตโนมัติ) |
| `T` | สลับ **ธีมสว่าง/มืด** (จำค่าไว้อัตโนมัติ) |
| `Y` | เปิด/ปิด **เสียงประกอบ** (สังเคราะห์ WebAudio ไม่มีไฟล์เสียง) |

**นาฬิกาจับเวลาผู้พูด** (คลิกเพื่อรีเซ็ต)

ปุ่มบนจอ (มุมขวาบน + แถบล่างในโหมดสไลด์) กดง่าย รองรับทีวี/ทัชสกรีน

## ฟีเจอร์

- **โหมด Scroll ↔ Slide** สลับได้ทันที (หน้ายาว scroll-reveal / เต็มจอ + จุดบอกตำแหน่ง + ตัวนับ)
- **อนิเมชันคุณภาพสูง:** split-word reveal หัวข้อ, reveal เมื่อเลื่อนถึง, count-up ตัวเลขสถิติ
- **มินิไทม์ไลน์อินเทอแรกทีฟ** เผยทีละจุด + เส้นเวลา **ไหลตามตำแหน่งสกรอลล์** (ค่อย ๆ ลากเส้นขณะอ่าน)
- **ภาพรวมเชิงตัวเลข** (by-the-numbers) + **อ้างอิงที่มา** ใต้สถิติแต่ละจุด
- **ตาชั่งสองด้าน** เปรียบเทียบผลงาน vs ข้อวิจารณ์ + **คำถามชวนคิด** ก่อนสรุป
- **Speaker notes** โน้ตผู้พูดต่อสไลด์ ซ่อนจากผู้ชม + **นาฬิกาจับเวลาผู้พูด**
- **Quiz** ปรนัยท้ายเรื่อง เฉลย + คำอธิบายทันทีเมื่อเลือก
- **ธีมสว่าง/มืด**, **deep-link** ราย slide ผ่าน URL hash (`...#policies`), **หน้าต่างคีย์ลัด**
- **Color-coded** ตามหมวด: ข้อมูลทั่วไป (น้ำเงิน) · ผลงาน (เขียว) · ข้อวิจารณ์ (แดงดิน) · บทเรียน (หญ้าฝรั่น)
- **Responsive** + รองรับ `prefers-reduced-motion` + **พิมพ์/Export PDF** เป็นแฮนด์เอาต์ 1 สไลด์/หน้า

### ★ ฟีเจอร์เสริมชุดใหม่ (19 อย่าง)

- **เปิดเรื่อง:** จักรอโศกหมุนตอนโหลด · ฮีโร่พารัลแลกซ์ + Ken Burns · สไลด์ **วาระ (agenda)** · แถบความคืบหน้าไล่สีไตรรงค์
- **ภาพช่วยเข้าใจ:** แผนที่อินเดีย (กุจราต→ทั้งชาติ) · บล็อกคน “1.4 พันล้าน ≈ 20× ไทย” · กราฟเส้น UPI (ภาพประกอบ) · แฟลชสีตามหมวด
- **เนื้อหารอบด้าน:** เลนส์ภาวะผู้นำต่อสไลด์ · ข้อโต้แย้งใต้ผลงานแต่ละชิ้น · **อภิธานศัพท์** แตะดูคำอธิบาย · **แหล่งอ้างอิง** ย่อเป็น ⓘ แตะ/โฮเวอร์
- **อินเทอแรกทีฟ:** **โพล** ผู้ฟัง · **ตาชั่งลากได้** (drag/ปุ่ม/คีย์บอร์ด) · คะแนน quiz + การ์ดผลดาวน์โหลดได้ · **กล่องสะท้อนความคิด** สะท้อนไปสไลด์ปิดท้าย
- **ปิดเรื่อง & ระบบ:** สไลด์ปิดท้ายชวนคิด · **เสียงประกอบ** (WebAudio, ปุ่ม Y) · **ซิงก์หลายหน้าต่าง** (BroadcastChannel — รีโมตแบบออฟไลน์)

> ทั้งหมดยังคงทำงาน **ออฟไลน์ 100%** ไม่มีไฟล์เสียง/ภาพภายนอก เคารพ `prefers-reduced-motion` และพิมพ์เป็น PDF ได้

### ★★ ชุดเสริมสำหรับการนำเสนอจริง (4 อย่าง)

- **หน้าต่างผู้นำเสนอ (`P`):** เปิดหน้าต่างที่สอง แสดง *สไลด์ปัจจุบัน + สไลด์ถัดไป + โน้ตผู้พูด + นาฬิกาจับเวลา* และปุ่มเดินสไลด์ — ซิงก์กับหน้าต่างผู้ชมผ่าน BroadcastChannel (หน้าต่างผู้ชมยังสะอาด)
- **อ่านออกเสียงไทย (`V`):** Web Speech API อ่านเนื้อหาแต่ละสไลด์เป็นเสียงไทย (ใช้เสียงไทยของระบบถ้ามี) — ช่วยการเข้าถึง ไม่มีไฟล์เสียง
- **สไลด์แหล่งอ้างอิง:** สร้างอัตโนมัติจากการอ้างอิงทุกจุด พร้อมป้าย *ยืนยันได้ / โดยประมาณ / ภาพประกอบ* — ตอกย้ำความโปร่งใสเชิงวิชาการ
- **สรุปของฉัน (PNG):** ปุ่มบนสไลด์ปิดท้าย รวม *คะแนน quiz + ผลโหวต + สิ่งที่จะนำไปใช้ + 3 บทเรียน* เป็นแฮนด์เอาต์หน้าเดียวดาวน์โหลดได้

> หมายเหตุ: การซิงก์หน้าต่างผู้นำเสนอใช้ BroadcastChannel (same-origin) — ทำงานได้ดีเมื่อเสิร์ฟผ่าน http(s) และบนเดสก์ท็อปส่วนใหญ่; ถ้าเบราว์เซอร์จำกัด origin ของ `file://` หน้าต่างผู้นำเสนอจะทำงานแบบสแตนด์อโลน (เดินสไลด์ของตัวเองได้)

## โครงสร้างโปรเจกต์

```
src/
  content.js      ← เนื้อหาทั้งหมด (ภาษาไทย) — แก้ที่นี่ที่เดียว
  styles.css      ← ดีไซน์ + อนิเมชัน
  app.js          ← ตรรกะอินเทอแรกทีฟ (vanilla JS, ไม่มี framework)
  template.html   ← โครง HTML (มี placeholder)
fonts/            ← ฟอนต์ไทย woff2 (Noto Sans/Serif Thai) — cache ไว้ build offline
scripts/
  fetch-fonts.sh  ← ดาวน์โหลดฟอนต์ (รันครั้งเดียวตอนต่อเน็ต)
build.js          ← ประกอบทุกอย่าง + ฝังฟอนต์ base64 → index.html
index.html        ← ★ ผลลัพธ์ self-contained (commit ไฟล์นี้ไปใช้ได้เลย)
```

## แก้ไขเนื้อหา / Build ใหม่

1. แก้ข้อความใน **`src/content.js`** (ทุกสไลด์ ไทม์ไลน์ สถิติ quiz โน้ตผู้พูด อยู่ที่นี่)
2. รัน build (ต้องมี Node.js):

```bash
node build.js
```

จะได้ `index.html` ใหม่ — รันได้ออฟไลน์ (ใช้ฟอนต์ที่ cache ใน `fonts/`)

ถ้ายังไม่มีฟอนต์ใน `fonts/` ให้ดึงครั้งแรก (ต้องต่อเน็ต):

```bash
bash scripts/fetch-fonts.sh && node build.js
```

### รูปพอร์เทรตจริงของโมดี

สไลด์แรกใช้ภาพถ่ายจริงจาก **`assets/modi.jpg`** ซึ่ง `build.js` จะฝังเป็น base64 อัตโนมัติ
ถ้าลบไฟล์นี้ออก ระบบจะกลับไปใช้ภาพ placeholder (ธง + เงา) แทน

อยากเปลี่ยนรูป: วางไฟล์ใหม่เป็น `assets/modi.jpg` แล้วรัน `node build.js`
